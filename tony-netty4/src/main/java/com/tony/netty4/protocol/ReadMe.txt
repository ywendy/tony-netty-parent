Netty 私有协议栈：

	1、通信模型
		
		|       |	(1)握手请求				|		|
        |       |-------------------------->|		|
        |       |	(2)握手应答            			|		|
        |       |<--------------------------|		|
        | Netty |   (3)发送业务消息        			| Netty |
        | 客户端      |-------------------------->| 服务端      |         
        |       |	(4)发送心跳消息        			|		|
        |       |<--------------------------|		|
        |       |	(5)发送心跳消息        			|		|
        |       |---------------------------|		|
        |       |	(6)发送业务消息        			|		|
        |       |<--------------------------|		|
        |       |...........................|		|
        |       | 关闭连接 <------->关闭连接...	|		|	
       
      ##具体步骤：
      	(1)Netty 协议栈客户端发送握手请求，携带节点ID等有效身份认证信息
      	(2)Netty 协议栈服务器对握手请求消息进行合法性校验，包括节点ID有效性校验，节点重复登录校验以及IP地址合法性校验，
      	         校验通过后，返回登录成功的握手应答消息.
      	(3)链路建立成功后，客户端发送业务消息
      	(4)链路成功后，服务端发送心跳信息
      	(5)客户端发送心跳信息
      	(6)服务端发送业务消息
      	(7)服务端推出，服务端关闭连接，客户端感知对方关闭连接后，被动关闭客户端连接
